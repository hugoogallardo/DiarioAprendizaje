Tema 1 GIT

Introducción a GIT

Git es un sistema de control de versiones distribuido que permite a los desarrolladores gestionar y hacer seguimiento a los cambios en el código fuente de un proyecto. Fue creado por Linus Torvalds en 2005, originalmente para el desarrollo del núcleo de Linux. Hoy en día es una de las herramientas de control de versiones más populares en el desarrollo de software.

Características clave de Git


  • Distribuido: A diferencia de los sistemas centralizados, cada copia de un repositorio Git es un repositorio completo en sí mismo. Esto significa que cada usuario tiene una copia completa de la historia y el estado del proyecto en su propia máquina.
  • Control de versiones: Git guarda un registro de todos los cambios realizados en los archivos del proyecto a lo largo del tiempo, permitiendo a los desarrolladores ver el historial de modificaciones, revertir cambios, crear ramas y fusionar código fácilmente.
  • Ramas y fusión: Permite crear ramas (branches) para trabajar en diferentes características o versiones de manera aislada y luego fusionarlas (merge) en la rama principal cuando están listas.
  • Alto rendimiento: Git está diseñado para ser rápido y eficiente en el manejo de grandes proyectos, gracias a su capacidad para manejar cambios incrementales y gestionar múltiples ramas sin grandes pérdidas de rendimiento.
  • Colaboración: Git facilita la colaboración entre desarrolladores mediante el uso de plataformas como GitHub, GitLab o Bitbucket, que permiten alojar repositorios de Git en la nube.

¿Para qué sirve Git?

Git es una herramienta esencial para el desarrollo de software moderno, ya que mejora la colaboración, la gestión del código y la calidad del producto final.En el anterior punto te explico algunas de sus funciones y beneficios más importantes.

Comandos de configuración de GIT

git --version > Nos dice la versión actual de GIT.
git config --list > Nos dice la configuración actul de GIT.
git config --global user.name > Coloca el nombre de usuariuo, donde en name debes poner tu nombre.
git config --global user.email > Colaca el correo electrónico, donde en email debes poner tu correo electrónico.

Comandos Iniciales de GIT

git help (comando) > Nos salta una ayuda para el comando o sintaxis de lo que quieres hacer.
git init > Inicializa un repositorio GIT

Cambios GIT

git add (archivo) > Añade un archivo al área de preparación.
git add . > Añade todos los archivos del ára de preparación.
git add .html > Añade todos los archivos con extensión html al área de preparación.
git commit -m "mensaje" > Confirma los cambios con un mensaje.
git commit --ammend -m "mensaje" > Agrega cambios al último commit.
git push > Sube los cambios al repositorio remoto.
git push -u origin master > Sube los cambios al repositorio remoto.

Empezando a funcionar

git status > Muestra el estado actual del repositorio.
git log --oneline > Nos muestra el historial de commits en un única línea.
git log > Aquí lo realiza pero de manera extendida.
git diff > Muestra las diferencias entre el directorio de trabajo y el área de preparación.
git diff --staged > Muestra las diferencias entre el área de preparación y el último commit.
git remote > Nos muestra una lista con los nombres remotos que hayamos dicho.
git remote -v > Nos muestra la URL de los repositorios remoto.
git remote add origin > Añade un repositorio remoto.
git remove origin > Elimina un repositorio remoto.
git remote show origin > Nos muestra información detallada de un repositorio remoto.
git remote rename origin (nuevonombre) > Renombra un repositorio remoto.
git branch -M main > Cambia el nombre de la rama master main.
git branch > Nos muestra las ramas locales.
git branch -a > Nos muestra todas las ramas.
git branch -d (nombre rama) > Elimina rama.
git branch -m (nombre rama) > Renombra una rama.

Rollback

git checkout (nombre rama) > Cambia a la rama indicada.
git checkout -b (nombre rama) > Crea una rama y la cambia a ella.
git reset HEAD (archivo) > Quita los archivos de staged y los devuelve al estado anterior.
git reset --soft (hash) > Mueve el HEAD al commit especificado.
git reset --hard (hash) > Mueve el HEAD al commit especificado y borra los cambios en el directorio de trabajo.
git reset --mixed (hash) > Mueve el HEAD al commit especificado y borra los cambios en el área de preparación.
git revert (hash) > Crea un nuevo commit que revierte los cambios de un commit especificado.
git revert HEAD > Crea un nuevo commit que revierte los cambios de un commit especifico.
git revert HEAD~2 > Crea un nuevo commit que revierte los cambios de los dos últimos commits.

Etiquetas

git tag > Muestra las etiquetas
git tag -a v1.0 -m "Versión 1.0" > Crea una etiqueta anotada.
git tag -d v1.0 > Elimina una etiqueta.

Crear,Borrar ramas

git branch > Muestra las ramas locales.
git branch (nombre rama) > Crea una rama.
git branch -a > Muestra todas las ramas.
git branch -d (nombre rama) > Elimina una rama.
git branch -m (antigui nombre) ( nuevo nombre) > Renombra una rama.

IDs de los commits del repositorio local

git log > Muestra los IDs de los commits del repositorio local.
git log -p > Incluye el historial de las confirmaciones, incluyendo todos los archivos y sus cambios.
git log --oneline > Muestra los IDs de los commits en una sola línea.
git log --oneline --all > Muestra los IDs de los commits de todas las ramas en una sola línea.

Moviimiento entre ramas

git checkout (nombre rama) > Cambia a la rama especificada.
git checkout -b (nombre rama) > Crea una rama y cambia a ella.
git checkout > Volver a cualquier version anterior en base al hash.

Fusionar Ramas

git merge (nombre rama) > Fusiona la rama especificada con la rama actual.
git merge --abort > Aborta la fusión de ramas.
git log --merge > Muestra la información de los 2 commits que han generado el conflicto.

GIT Clone

git clone (repositorio remoto) (directorio local) > Se usa para copiar un repositorio remoto en local.
git clone --branch (tag) (repositorio remoto) > Se usa para copiar únicamente la version de la tag indicada.
git clone --branch --single-branch (branch rama) (repositorio remoto) > Se usa para copiar únicamente la rama indicada.

Traer Cambios

Git pull > Trae los cambios del repositorio remoto y los fusiona con el repositorio local.
Git Merge > Fuisiona los cambios de una rama con la rama actual.
Git Fetch > Trae los cambios del repositorio remoto pero no los fusiona con el repositorio local.

Bajar datos del repositorio remoto

git pull > Trae los cambios del repositorio remoto y los fusiona con el repositorio local.
git pull origin main > Trae los cambios del repositorio remoto y los fusiona con el repositorio local.
git pull --verbose > Muestra información detallada de los cambios que se están trayendo.
